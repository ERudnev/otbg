# Contract

## Что считаем внешним контрактом

Симуляция имеет два внешних интерфейса:

1. **Входной контракт** — текстовый сценарий команд.
2. **Выходной контракт** — поток событий симуляции.

## Входной контракт

Нельзя менять формат сценария:
- имена команд;
- порядок и смысл аргументов;
- базовую модель "файл со сценарием -> выполнение симуляции".

Текущие команды:
- `CREATE_MAP`
- `SPAWN_SWORDSMAN`
- `SPAWN_HUNTER`
- `MARCH`

Важно:
- внутренний движок может меняться как угодно;
- парсер и runtime можно реорганизовывать;
- но существующий сценарий должен продолжать запускаться без изменений.

## Выходной контракт

Нельзя ломать формат событий:
- строковый формат лога;
- смысл существующих событий;
- базовую модель "`[tick] EVENT field=value ...`".

Текущие события:
- `MAP_CREATED`
- `UNIT_SPAWNED`
- `MARCH_STARTED`
- `MARCH_ENDED`
- `UNIT_MOVED`
- `UNIT_ATTACKED`
- `UNIT_ABILITY_USED`
- `UNIT_DIED`

## Особо важное ограничение

`UNIT_ATTACKED` должен отражать **суммарный урон**, который один юнит нанес другому в рамках одного наблюдаемого действия.

Это критично для:
- обычных атак;
- способностей;
- яда;
- комбинированных эффектов вроде poison + rending.

## Что можно менять свободно

Можно менять:
- устройство `Game` / `Core` / `Features`;
- `World`, `Unit`, `Map` и их роли;
- внутреннюю модель эффектов;
- способ исполнения хода;
- внутренние команды, очереди, processors, ECS и т.д.;
- способ хранения состояния.

## Что можно расширять

Можно расширять выходной слой **только если появился новый наблюдаемый факт**:
- новый тип действия;
- новый вид эффекта;
- новый объект мира, который должен быть виден наружу.

Но:
- не стоит вытаскивать наружу внутренние детали движка;
- не стоит плодить события, которые отражают только внутреннюю кухню.

## Архитектурный вывод

Наружу должен быть стабилен **контракт симуляции**.

Внутри можно радикально менять реализацию, если:
- старые сценарии продолжают работать;
- старые ожидаемые события не ломаются;
- новые механики выражаются без хака и без утечки внутренних деталей.
